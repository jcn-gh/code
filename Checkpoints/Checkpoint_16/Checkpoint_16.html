Preguntas

  • ¿Por qué utilizamos rutas de guardia?

    Las rutas de guardia son una forma de controlar el acceso a ciertas rutas en una aplicación. Se utilizan para asegurarse de que solo los usuarios autorizados puedan acceder a determinadas partes de la aplicación.

    Las rutas de guardia se pueden implementar utilizando componentes de enrutamiento como React Router. Estos componentes permiten definir rutas y especificar qué acciones deben realizarse cuando se accede a una ruta determinada.

    Por ejemplo, en una aplicación de autenticación, se puede utilizar una ruta de guardia para asegurarse de que solo los usuarios autenticados puedan acceder a la página de inicio. Si un usuario no está autenticado, se puede redirigir a una página de inicio de sesión o mostrar un mensaje de error.

    Aunque el uso de rutas de guardia puede introducir una cierta complejidad adicional en la aplicación y es posible que se requiera más código y lógica para implementarlas y mantenerlas correctamente, sus beneficios lo compensan:

    • Permiten controlar el acceso a diferentes partes de la aplicación. Esto puede ser útil para aplicaciones que requieren autenticación o autorización, como aplicaciones de administración o aplicaciones de comercio electrónico.
    • Pueden ayudar a garantizar que solo los usuarios autorizados puedan acceder a ciertas partes de la aplicación. Esto puede mejorar la seguridad y la privacidad de la aplicación.
    • Pueden mejorar la experiencia del usuario al proporcionar una navegación más fluida y evitar que los usuarios accedan a contenido no deseado o no autorizado.
    • Pueden ayudar a organizar el código y la estructura de la aplicación. Se pueden definir rutas específicas y asociar acciones o componentes correspondientes a cada una de ellas.

    Consejos para grantizar una implementación eficaz y segura.

    • Separación de responsabilidades: Es importante separar la lógica de las rutas de guardia de la lógica de los componentes de la aplicación. Esto ayuda a mantener un código más limpio y fácil de mantener.
    • Centralización de la lógica de autorización: Es recomendable centralizar la lógica de autorización en un solo lugar. Esto facilita la gestión y mantenimiento de las reglas de autorización en la aplicación.

    • Uso de middleware: Puede ser útil utilizar middleware para manejar la lógica de autorización en las rutas de guardia. Esto puede simplificar la implementación y permitir reutilizar la lógica en diferentes rutas.

    • Protección de rutas sensibles: Asegúrate de proteger las rutas sensibles de forma adecuada. Por ejemplo, las rutas que requieren autenticación deben estar protegidas para evitar el acceso no autorizado.
    • Manejo del redireccionamiento: Define de manera clara, y consistente, cómo manejar las redirecciones en caso de que un usuario no esté autorizado para acceder a una determinada ruta. Puedes redirigir al usuario a una página de inicio de sesión o mostrar un mensaje de error.
    • Pruebas: Asegúrate de escribir pruebas para las rutas de guardia. Esto te ayudará a verificar que la lógica de autorización funciona como se espera y a detectar posibles problemas de seguridad.

    Siguiendo estas buenas prácticas, puedes implementar de manera efectiva rutas de guardia en tu aplicación React y mejorar la seguridad y la experiencia del usuario.

    Componente Redirect

    Aunque pueda paracer un poco extraño que un componente nos redireccione al intentar acceder a una página, no es ni más ni menos que lo que ocurre al intentar acceder a una página privada sin habernos autenticado primero: somos redirigidos a la página de inicio de sesión.

    Las entradas de la función PrivateRoute son el componente a mostrar si el usuario está autenticado y el resto de props (parámetros) pasados al componente.

    Se usa el hook useAuth para obtener el estado de autenticación del usuario y, basándose en el mismo, se muestra la ruta correspondiente. Si está autenticado el componente especificado es mostrado con las props pasadas al componente PrivateRoute y si no lo está, el usuario es redirigido a la página principal.

    Consulta la documentación oficial de las librerías de React, React Router v6, y JSON Web Token para conocer más detalles.

    Hook useHistory

    A parte de los anteriores, quizás sea useHistory el gancho más útil. Podemos llamarlo en la parte superior de cualquier componente declarado dentro de nuestro componente de enrutador y recuperar datos del historial con history, que incluyen información como la ubicación asociada a nuestro componente mediante history.location.

    Además, history incluye dos métodos, history.push y history.replace, que nos permiten dirigir al usuario a diferentes páginas de nuestra aplicación. La diferencia entre ambos es que con el último se borrará el historial tras realizar la navegación (como cuando el usuario cierra la sesión).

    Hook useLocation

    Cuando solo necesitamos los datos de ubicación podemos ahorrar memoria, y ganar algo de agilidad, usando use.Location en lugar de history.location.

    Recomendaciones

    • Organizar las rutas: Mantén tus rutas organizadas y separadas en archivos o módulos para que sea más fácil de entender y mantener.
    • Utilizar componentes de enrutamiento: Utiliza componentes de enrutamiento como BrowserRouter, Route, Switch y Link para manejar el enrutamiento en tu aplicación.
    • Utilizar rutas dinámicas: Utiliza rutas dinámicas para permitir la navegación a través de datos, como /users/:id para mostrar detalles de un usuario específico.
    • Utilizar parámetros de ruta: Utiliza parámetros de ruta para capturar y pasar datos a través de las rutas, como /products/:id para mostrar detalles de un producto específico.
    • Utilizar protección de rutas: Utiliza protección de rutas para permitir el acceso, por ejemplo, solo a usuarios autenticados.
    • Utilizar carga progresiva: Utiliza carga progresiva para cargar solo los componentes necesarios para cada ruta, lo que puede mejorar el rendimiento de la aplicación.
    • Si tu aplicación es muy simple y no necesita un enrutamiento complejo, podrías considerar no usar React Router y manejar el enrutamiento manualmente con componentes de React.
    • Si estás trabajando en un proyecto existente que ya tiene un enrutamiento implementado y no deseas cambiarlo, podrías optar por no usar React Router y seguir utilizando el enrutamiento existente.
    • Si estás trabajando en un proyecto que tiene requisitos específicos de rendimiento y escalabilidad, y React Router no se ajusta a esos requisitos, deberías considerar otras opciones de enrutamiento.

    Beneficios del enrutador React

    • Proporciona una forma de manejar el enrutamiento en la aplicación, como navegar entre diferentes vistas o páginas.
    • Es componible y adaptativo.
    • Agregar enlaces fácilmente después de diseñar la página web
    • Ofrece una serie de características avanzadas, como la gestión de parámetros de ruta, el enrutamiento basado en rutas anidadas y la protección de rutas.

    En contra

    • Puede aumentar la complejidad de la aplicación si se utiliza en exceso, especialmente si hay muchas rutas y componentes asociados.
    • Puede dificultar la navegación en la aplicación si las rutas están demasiado anidadas o si hay demasiados parámetros de ruta.
    • Puede generar un código más difícil de entender y mantener si no se utiliza correctamente, especialmente si hay muchas rutas y componentes asociados.
    • Puede agregar un tiempo de carga adicional a la aplicación si se utiliza en exceso, especialmente si hay muchas rutas y componentes asociados.
    • Puede generar un tamaño de paquete más grande para la aplicación si se utiliza en exceso, especialmente si hay muchas rutas y componentes asociados.
    • Puede dificultar la indexación de la aplicación por parte de motores de búsqueda si las rutas son demasiado dinámicas o si hay demasiados parámetros de ruta.

    Conclusión

    El uso de React Router depende del contexto de tu proyecto y de tus necesidades específicas. Si React Router se adapta a tus necesidades y te brinda las funcionalidades que necesitas, es una buena opción. Sin embargo, si hay otras alternativas que se ajusten mejor a tus requisitos, deberías considerar utilizarlas.

  • ¿Qué es una solicitud POST?

    Es un método HTTP que envía datos al servidor. Es decir, significa crear o actualizar un recurso en el Servidor.

    La representación del cuerpo del mensaje se indica mediante el encabezado Content-Type (tipo de contenido). Dependiendo de este valor, el servidor tiene diferentes expectativas en cuanto a cómo se formatea el cuerpo del mensaje.

    application/x-www-form-urlencoded

    Este es el tipo de contenido predeterminado, que especifica la codificación en pares clave-valor. Cada par está delimitado por el carácter & y las asignaciones se indican con el signo igual =. Los valores no alfanuméricos se escapan mediante codificación porcentual. Se pueden utilizar varios pares clave/valor en una sola línea.

    multipart/form-data

    En un formulario de varias partes, cada valor se envía como un bloque de datos y se utiliza un delimitador para separar cada parte. El delimitador entre partes se llama Boundary (Límite) y se especifica como parte del encabezado Content-Type. Las claves se especifican en el encabezado Content-Disposition (disposición del contenido) de cada parte del cuerpo del mensaje.

    text/plain

    No existe un formato prescrito para el cuerpo del mensaje y se espera que el servidor lo procese en consecuencia.

    El código de estado HTTP devuelto por el servidor después de recibir una solicitud HTTP POST variará según el resultado del procesamiento interno.

    El método HTTP POST es similar a HTTP PUT, donde se puede utilizar para crear un recurso, aunque PUT es idempotente, mientras que una solicitud POST idéntica no lo es, lo que significa que la misma solicitud POST puede tener diferentes efectos dependiendo de cómo se maneje en el servidor. Por ejemplo, si se envía una solicitud POST para crear un nuevo registro en una base de datos, esa misma solicitud POST podría crear un nuevo registro cada vez que se envíe, incluso si los datos son los mismos.

    En un método de solicitud POST, los datos del formulario se codifican en el cuerpo del mensaje. Ésta es una diferencia importante entre GET y POST.

    Ventajas de POST:

    • Los datos del formulario se envían utilizando el cuerpo del mensaje. Por lo tanto, es más seguro que GET.
    • No hay limitación en la longitud de los datos del formulario.
    • Sin restricciones sobre el tipo de datos. También se permiten datos binarios.

    Desventajas de POST:

    • No se puede marcar como favorito.
    • Una solicitud HTTP POST se puede almacenar en caché, pero solo si se incluye información sobre la actualización.

    Los usos comunes de las solicitudes HTTP POST son:

    • Enviar un bloque de datos a un recurso de procesamiento de datos, como la información ingresada por un usuario en un formulario HTML.
    • Modificar o agregar datos a un recurso existente.
    • Transferir un mensaje o artículo de blog para publicarlo en un foro.
    • Indicar al servidor que realice acciones como iniciar o detener tareas.
    • Crear un nuevo recurso.

    Es decir, utiliza GET para operaciones seguras como una solicitud de una página o consultar una base de datos. Si alguna información que envíes al servidor es confidencial, no utilice GET. Porque todo lo que envías al servidor es visible en la URL. Por ejemplo, información como nombre de inicio de sesión, contraseña, etc. no debe enviarse mediante GET.

    Para todas las demás operaciones utiliza el método POST.

  • ¿Qué es un Portfolio y porque es importante?

    Un portafolio es una colección de documentos destinado a mostrar la creación de trabajos en distintas áreas, ya sea en diseño, arquitectura o fotografía, exhibiendo el esfuerzo del profesional y sus habilidades.

    El portafolio sirve como una herramienta de recopilación, donde se recoge lo mejor que se ha producido a lo largo del tiempo.

    No es un sitio para mostrar la experiencia profesional, para eso está el CV, aquí únicamente se busca presentar los resultados a través de la muestra de diferentes productos.

    Con este recurso podrás organizar tu archivo personal para mostrar tu trabajo en la web de una manera más profesional y respaldar tu currículo con el mismo, ayudándote así a crear alianzas y contactos.

    Cómo crear un portafolio paso a paso

    Paso 1:

    Crea una carpeta, para añadir cada uno de tus trabajos más destacados dentro de la misma, y así tener un amplio panorama de lo que sí debes incluir y de lo que no en tu portafolio. Es importante que trates de ser lo más intuitivo posible y seleccionar lo mejor que hayas elaborado a lo largo de tu trayectoria.

    Paso 2:

    Organiza cada uno de tus proyectos cronológicamente o por categorías.

    Paso 3:

    Un resumen es la clave, recuerda que no queremos un álbum de fotos sino un verdadero “portafolio digital”, además, es importante que describas brevemente cada uno de ellos. Tu portafolio se verá mucho más armonioso y con la información necesaria.

    Paso 4:

    En internet encontrarás diversos sitios web para colocar tu portafolio, hay plantillas que te harán más llevadero el proceso de creación.

    Algunos aspectos que puedes tener cuenta:

    • Mantén la calidad de tus imágenes.
    • Generar un nombre de dominio para tu portafolio.
    • Portafolios en línea optimizados para Google.
    • Máxima velocidad y banda ancha ilimitada.
    • Personalización de las plantillas.
    • Disponibilidad 24/7 en la web para que tus visitantes puedan verlo cuando deseen.
    Paso 5:

    Solo te queda darte a conocer después de que hayas puesto todo tu tiempo, empeño y creatividad en crear un portafolio.

    Asimismo, puedes cargar el portafolio en diferentes páginas web de empleo con aquellas vacantes relacionadas a tu profesión o enviarlo vía correo electrónico a aquella empresa con la que siempre has querido trabajar, no olvides que el límite lo pones tú.

    Recordatorio

    Un portfolio es importante para mostrar tus habilidades y proyectos a posibles empleadores o clientes. Puede demostrar tu experiencia, creatividad y capacidad para resolver problemas a través de ejemplos concretos.

  • ¿Cuáles son algunas cosas que puede hacer para prepararse para las entrevistas de trabajo?

  • ¿Que es Linkdn y para que se utiliza?

    Cuántas veces escuchaste: ¿te hiciste un perfil en Linkedin? ¿estás en Linkedin? ¿lo encontraste en Linkedin?. Y podemos seguir. Linkedin es una red social profesional orientada al ámbito empresarial.

    Entonces… ¿qué es y cómo funciona Linkedin?

    Linkedin es una plataforma social y profesional, una red que está principalmente orientada a empresas, negocios y empleos.

    La idea es que, a partir de los perfiles que se crean los usuarios (experiencia laboral, virtudes de los usuarios, etc), la web conecta a millones de empleados y de empresas en todo el mundo. De esta manera, se busca poner en contacto a determinadas personas que se encuentran buscando nuevas oportunidades en el mercado laboral y profesional.

    Es una plataforma que cuenta ingresos en base a suscripciones de diferentes usuarios, publicidad en la plataforma y en las distintas soluciones para la selección del personal. Por ello, en diciembre de 2016, Microsoft se convirtió en el propietario de esta plataforma considerada, a nivel mundial, la comunidad más profesional de todas.

    Esta red, a diferencia de las que todos conocemos, intenta ser un nexo en el espacio laboral y profesional. Es muy importante que tanto el pequeño emprendedor o como grandes empresas tengan presencia en esta plataforma.

    Entonces… ¿para qué sirve Linkedin?

    • Mostrar cuáles son nuestras aptitudes profesionales y hacer Networking:

      Es importante contar con una amplia red de contactos de profesionales con los que podamos negociar y establecer relaciones comerciales. De ahí la importancia de contar con una página web (o por lo menos tener un Blog).

      Por eso es importante un término conocido, en el mundo de los negocios, como Networking, que hace referencia a una actividad socioeconómica, donde los profesionales y/o emprendedores se juntan para crear relaciones empresariales, desarrollar nuevas oportunidades de negocios, compartir su información e intentar atraer clientes potenciales.

    • Encontrar el empleo deseado en base a nuestras aptitudes:

      Muchas veces, cuando estamos buscando trabajo, entramos en Linkedin para saber cuáles son los perfiles que se están buscando en el empleo que estemos buscando. Así esta plataforma te informa de los perfiles y aptitudes que necesitan las empresas. En los perfiles de las empresas podemos encontrar sus ofertas de empleo. Hay que estar atentos en el día a día para ver si lo que buscan encajan con tu perfil.

      A veces las empresas buscan directamente a la persona que necesitan, y con un perfil en Linkedin y un CV (Currículum Vitae) hay más oportunidades de que te conozcan.

    • Encontrar nuevos proveedores para nuestro negocios:

      Es fácil encontrar en el buscador de Linkedin profesionales que podrían llegar a ser nuestros futuros proveedores.

    • Encontrar nuevos clientes en el mercado laboral:

      Lo mismo que ocurre con los proveedores, ocurre con los clientes: podemos relacionarnos con departamentos o secciones de empresas que estén buscando los servicios que nosotros ofrezcamos.

    • Aumentar el tráfico de nuestra página web:

      Es una buena manera de aumentar el tráfico de nuestro sitio web, blog o de lo que dispongamos. Como ocurre en las redes sociales, si generamos contenido para compartirlo, las personas interesadas podrán terminar en nuestro sitio web.

    Misión y Visión de Linkedin

    Su idea principal es conectar a los profesionales de todo el mundo para que terminen siendo más productivos y ayudarlos a alcanzar las metas laborales que se hayan propuesto.

    Su objetivo es generar nuevas oportunidades económicas para los miembros del mercado laboral.

Menciona cuáles son tus objetivos profesionales, y cómo quieres conseguirlos.

    • Deja que las fotos te representen

      ¡Tranquilo, fiera! Eso no quiere decir que tengas que salir… de cualquier manera. La foto del perfil y del banner deben ser profesionales, pero con más gracia que la del DNI. Piensa en cómo te vistes para una entrevista laboral, ¿lo tienes? Pues igual, opta por una imagen de hombre de negocios elegante, contemporáneo y urbano.

      Tu banner es lo primero que se verá al entrar en tu perfil. Debe ser atractivo, añade color y deja que tu creatividad hable por tí. Y ¿por qué no? Aprovecha para anunciarte y enumera tus mayores logros.

    • Resalta tu experiencia laboral

      Hazlo de tal forma que cuando las empresas lean este apartado, tengan la impresión de que tus habilidades son equiparables a las de un genio o que gestionas proyectos como un director de orquesta.

      Ve directo al grano y resume tu experiencia en puntos, en vez de en párrafos. Describe 4 o 5 funciones por empleo y deja al reclutador con ganas de querer conocer más de ti.

      Tus logros y resultados sin números no son nada. Pon números y porcentajes como si fueras profesor de Matemáticas, recuerda que deben cantar tus logros, no susurrarlos.

      Siempre es aconsejable personalizar el CV a cada oferta laboral. Haz lo mismo aquí. Cada puesto debe verse como un traje hecho a medida. Resalta las habilidades y logros que corresponden a tus objetivos profesionales actuales.

    • Muestra tu formación académica con orgullo

      Piensa que los estudios son los cimientos de la fortaleza que estás construyendo. Lo que has estudiado proporciona una visión de los conocimientos que tienes, así que, para cada título, comparte una de las aptitudes profesionales que hayas conseguido. Aprovecha para agregar las actividades extraescolares y clubes de estudio que han formado parte de tu formación académica.

    • Destaca tus habilidades al máximo

      Basándose en la información profesional que hayas puesto en las secciones previas, Linkedin te recomendará una selección de hard skills (habilidades/competencias técnicas) relativas a tu industria. A su vez, podrás manualmente buscar otras habilidades técnicas e incluso soft skills (habilidades sociales) que se adapten a tu perfil y añadir ¡hasta 16 habilidades y competencias!

      Coloca tus habilidades más relevantes y codiciadas primero. ¡Tus habilidades y competencias digitales deberían liderar el grupo!

      Consigue recomendaciones. Cuando tus conexiones respaldan tus habilidades, tu perfil automáticamente se llena de credibilidad.

    • Termina tu perfil de LinkedIn con proyectos y voluntariados

      Al crear tu perfil puedes incluir una gran variedad de información adicional para demostrar que las últimas secciones ¡no son menos que el resto de tu página!

      Incluir un portfolio con tus proyectos te ayudará a diferenciarte de los otros solicitantes de empleo y a demostrar tu valor de una manera más dinámica y atractiva.

      Tus actividades como voluntario muestran tu pasión, tu empatía y tu afán de contribución a un bien mayor.
